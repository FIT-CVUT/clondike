46 /** @defgroup tcmi_man_class tcmi_man class
47  *
48  * @ingroup tcmi_managers_group
49  *
50  * This template <<singleton>> class implements a generic cluster
51  * node manager - TCMI manager. It provides generic functionality to control
52  * migration managers.
53  *
54  * Each migration manager is associated with one registered PEN. The
55  * user controls the cluster node(PEN,CCN or both) by manipulating
56  * \link tcmi_ctlfs.c TCMI control file system \endlink. The
57  * directory/file layout can be divided into three groups of
58  * files/directories based on their maintainers, which can be:
59  *
60  * - TCMI manager
61  * - TCMI migration manager
62  * - TCMI preemptive migration component
63  * - TCMI non-preemtive migration component
64  * - TCMI tasks
65  *
66  *
67  * Description of a particular layout is detailed at \link
68  * tcmi_ccnman_class TCMI CCN manager \endlink and \link
69  * tcmi_penman_class TCMI PEN manager \endlink
70  *
71  *
72  * There is an upper bound on the number of registered nodes. This is
73  * a very common approach in UNIX design as it prevents problems with
74  * overloading the CCN. This parameter is currently set at compile
75  * time. but might be made configurable via ctlfs from user space.
76  *
77  * The core datastructure for keeping track of migration
78  * managers(=registered cluster nodes) is \link tcmi_slotvec_class a
79  * slot vector \endlink.
80  *
81  * The TCMI manager carries a unique identifier that is generated
82  * randomly upon initialization. This ID then identifies a node in
83  * cluster.
84  *
85  * @{
86  */
87
