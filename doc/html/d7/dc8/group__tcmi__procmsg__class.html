Detailed Description

This class represents a generic process message.

Process message, in addition to regular migration message, carries a target process PID. This PID will be used by the receiving party for message delivery. The only functionality provided by this class is to send/receive the process ID and notify the specific procmsg via send/receive/free operations.

<<public>> Initializes the process message for transferring.

Similar to rx version - the message operations specific to a particular process message class are stored in the instance, Regular tcmi message is supplied with generic proc message operations that handle the polymorphism further.

Parameters

msg_id- ID of a particular message type
*msg_ops- pointer to a particular process message specific operations
dst_pid- destination process ID - filled in the header
enfoce_migmode- do we require switch to migration mode upon recieving of this message?
*transactions- storage for the new transaction(NULL for one-way messages)
resp_msg_id- response message ID - required for the transaction if the transaction expires(NULL for one-way messages)
