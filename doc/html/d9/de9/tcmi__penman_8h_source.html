48 /** @defgroup tcmi_penman_class tcmi_penman class
49  *
50  * @ingroup tcmi_man_class
51  *
52  * This <<singleton>> class implements the PEN TCMI manager
53  * component. It is responsible for registering the cluster node to
54  * CCN's specified by the user. Each migration manager communicates with one
55  * CCN. User controls the process execution node by manipulating \link
56  * tcmi_ctlfs.c TCMI control file system \endlink. The directory/file
57  * layout can be divided into three groups of files/directories based
58  * on their maintainers, which can be:
59  *
60  * - TCMI PEN manager
61  * - TCMI PEN migration manager
62  * - TCMI PEN preemptive migration component
63  * - TCMI PEN non-preemtive migration component
64  * - TCMI migrated process
65  *
66  * As we can see these are exact counterparts to the components on CCN.
67  *
68  \verbatim
69  ------------ T C M I P E N m a n a g e r --------------
70  pen/connect -> allows connecting to a CCN
71 
72  stop -> writing into this stops the whole framework, migrates all processes back
73  to their CCN's and terminates all migration managers.
74 
75  ----------- T C M I P E N m i g r a t i o n m a n a g e r s --------------
76  nodes/1/ -> each CCN where this PEN is registered has a separate directory, maintained
77  by its migration manager that is in charge of it.
78  2/
79  .
80  .
81  9/connections/ -> all connections of a migration manager
82  ctrlconn/ -> control connection
83  arch -> contains architecture of the connection
84  localname -> local socket name (address)
85  peername -> remote socket name (address)
86  state - current state of the migration manager
87 
88  ------------ T C M I P E N s h a d o w p r o c e s s e s --------------
89  mig/migproc/1300/ -> similar to /proc, but contains PID's of migrated processes
90  one directory per process
91  2001/
92  .
93  .
94  6547/migman->../../nodes/7 -> a symbolic link pointing to
95  its current CCN
96  remote-pid -> process PID at its original CCN
97  ------------ T C M I P E N p p m c o m p o n e n t --------------
98  emigrate-ppm-p -> writing a PID + PEN id pair into this file starts process
99  migration to the specified PEN.
100  migrate-home -> allows migrating a specified process back to CCN
101  ------------ T C M I P E N n p m c o m p o n e n t (not implemented) --------------
102  policy -> interface for the migration policy, the npm component
103  uses this interface to ask the migration policy for migration
104  decision.
105 
106  \endverbatim
107  *
108  *
109  * @todo InterPEN migration is not implemented.
110  * @{
111  */
