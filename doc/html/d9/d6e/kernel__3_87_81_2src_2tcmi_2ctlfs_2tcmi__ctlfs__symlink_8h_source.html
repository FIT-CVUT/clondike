37 /** @defgroup tcmi_ctlfs_symlink_class tcmi_ctlfs_symlink class
38  *
39  * @ingroup tcmi_ctlfs_entry_class
40  *
41  * This class provides a robust solution for symbolic link creation.
42  * A symbolic link can be made to any entry within the TCMI control
43  * file system. The link instantiator provides only the target entry
44  * object that the link should point to. The class itself then takes
45  * care of creating the pathname to the target on fly as follow link
46  * requests come. There are three VFS inode operations that the link
47  * must provide:
48  *
49  * - follow_link - supplies the target object pathname to the VFS
50  * - put_link - a notification form VFS that the target pathname is not
51  * needed anymore after follow_link operation.
52  * - readlink - copies the target pathname to the buffer supplied by VFS
53  *
54  * This class implements only the first 2 operations
55  * (tcmi_ctlfs_symlink_follow_link() and
56  * tcmi_ctlfs_symlink_put_link()). The third one uses VFS generic_readlink()
57  * that in turn invokes the custom tcmi_ctlfs_symlink_follow_link()).
58  *
59  * Each symbolic link instance retains a reference to the target entry.
60  * This reference is released upon symlink destruction.
61  *
62  * @{
63  */
64
