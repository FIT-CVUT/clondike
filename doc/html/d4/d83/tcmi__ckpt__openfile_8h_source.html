47 /** @defgroup tcmi_ckpt_openfile_class tcmi_ckpt_openfile class
48  *
49  * @ingroup tcmi_ckpt_class
50  *
51  * This is a helper <<singleton>> class used by the \link
52  * tcmi_ckpt_class checkpoint \endlink and provides functionality to:
53  *
54  * - write an information about an open file into the checkpoint
55  * - restore/reopen a file based on information in the checkpoint
56  *
57  * The checkpoint instance during checkpoint image creation passes
58  * each file along with its file descriptor and current file
59  * descriptor cache to this component. The are two separate cases:
60  * - the file information has already been written into the checkpoint -
61  * this is indicated by a file object being already present in the
62  * \link tcmi_fdcache_class file descriptor cache \endlink.
63  * In this case only a short information about a file is written
64  * into the checkpoint file - it's current file descriptor and the
65  * file descriptor that it's duplicating
66  * - the file information hasn't been written into the checkpoint
67  * (not present in the fd cache). In this case a full record
68  * is required:
69  * - file descriptor
70  * - current position in file
71  * - file flags used when opening the file
72  * - file access mode
73  * - file type(regular/directory/char/block device/pipe/socket/symlink)
74  * - full pathname
75  *
76  *
77  * Initial implementation supports checkpointing of regular files and
78  * directories only. There is a limited support when checkpointing
79  * open character devices - seek operation is omitted. Future
80  * implementation when restoring the open files should consider the
81  * circumstances and decided to reopen the original file or special
82  * proxy file. The main purpose of the proxy file will be to forward
83  * all requests to the original node.
84  *
85  * @todo Implement handling of special files (character devices/block
86  * devices, pipes, sockets)
87  *
88  * @{
89  */
